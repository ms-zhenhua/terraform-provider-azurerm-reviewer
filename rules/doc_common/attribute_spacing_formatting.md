Attribute documentation must follow consistent spacing rules to maintain readability and professional appearance. Each attribute should have exactly one space after the attribute name before the hyphen, one space after the hyphen, and one space after the requirement level (e.g., `(Required)`) before the description.

## good

```markdown
* `type` - (Required) The type of managed identity that should be configured on the Compute Fleet. The only possible value is `UserAssigned`.
```

## bad

```markdown
* `type` - (Required)  The type of managed identity that should be configured on the Compute Fleet. The only possible value is `UserAssigned`.

* `type` - (Required)The type of managed identity that should be configured on the Compute Fleet. The only possible value is `UserAssigned`.

* `type`- (Required)  The type of managed identity that should be configured on the Compute Fleet. The only possible value is `UserAssigned`.

* `type` -(Required)  The type of managed identity that should be configured on the Compute Fleet. The only possible value is `UserAssigned`.
```
