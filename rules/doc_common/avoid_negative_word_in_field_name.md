Avoid negative words in field names that obfuscate the purpose of the field. Use positive, clear naming conventions that directly express the intended behavior.

### good:
```markdown
* `storage_enabled` - (Optional) ....
* `user_upload_enabled` - (Optional) ....
* `logging_enabled` - (Optional) ....
* `encryption_enabled` - (Optional) ....
```

### bad:
```markdown
* `no_storage_enabled` - (Optional) ....
* `block_user_upload_enabled` - (Optional) ....
* `disable_logging_enabled` - (Optional) ....
* `not_encryption_disabled` - (Optional) ....
```
